# 타당성 검증 보고서

일자: 2026-02-26  
주제: 제안된 아이폰 멀티캠 파노라마 로드맵 대비 현재 프로젝트 상태 검증

## 1) 요약 결론

제안된 로드맵은 기술적으로 타당하지만, 단계적 추진이 전제되어야 합니다.  
현재 프로젝트는 **트랙1(오프라인 스티칭 MVP)** 를 중심으로 동작하며, 품질은 규칙 기반 보정 수준입니다.  
트랙2/트랙3의 AI 핵심 모듈은 아직 구현되지 않았습니다.

실무 판단:

- 오프라인 스티칭 확장 및 백엔드 고도화는 **Go**
- AI 자동 팬/줌은 뷰포트 파이프라인/탐지 baseline 이후 **조건부 Go**
- 라이브 실시간 목표는 현재 성능 기준 **No-Go**

## 2) 현재 프로젝트 역량 스냅샷

## 구현됨

- 오프라인 이미지/영상 스티칭 파이프라인
- 리포트 기반 관측성(`status`, `error_code`, stage 시간, sync/blend 메트릭)
- 영상 동기화: coarse + refine offset 추정
- 품질 휴리스틱: 적응형 seam-cut, 노출 보정, seam 경로 최적화
- 백엔드 잡 API/워커/스토리지 기본 구조
- 영상 프리셋 명령:
  - `video-10s`
  - `video-30s`
  - `video-full`

## 미구현

- AI 자동 팬/줌 정책 모듈(뷰포트 생성기)
- 스포츠 문맥 객체 탐지/추적 파이프라인
- 아이폰 라이브 스트림 수신/재연결/지터 버퍼/폴백
- AI 모델 기반 seam/고스팅 보정(현재는 규칙 기반 CV)
- 멀티 스포츠 정책 플러그인 구조

## 3) 최근 실행 근거

참조 리포트:

- `output/videos/video01_10s_report.json`
- `output/videos/video04_10s_report.json`
- `output/images/pair04_quality2_report.json`

관측 결과:

- 현재 샘플에서는 스티칭 성공
- 어려운 장면에서는 품질 경고 지속(`high_overlap_difference_seam_cut`)
- 처리속도는 실시간 미달:
  - 예시 `video01_10s`: 10초 콘텐츠 처리에 약 38.9초
  - 예시 `video04_10s`: 10초 콘텐츠 처리에 약 97.5초

의미: 현 스택은 오프라인 배치에는 적합하지만 라이브 운영에는 부적합.

## 5) 제안된 단계 로드맵 검증

제안된 순서는 적절합니다.

1. 오프라인 스티칭 baseline
2. AI 없는 팬/줌 렌더러
3. 그 위에 AI 정책 결합
4. 라이브 전환
5. AI 품질 보정 고도화

판단: 구현 의존성 순서와 일치하며 현실적인 접근입니다.

## 6) 현재 코드 기준 핵심 리스크

1. **실시간 격차**
   - 10초 클립 처리시간이 10초를 크게 초과
   - 라이브 목표에는 대규모 최적화(또는 GPU 경로) 필요

2. **기하 모델 한계**
   - 단일 전역 변환으로는 근거리 시차에 취약
   - 어려운 장면에서 seam/고스팅 가시성 유지

3. **AI 범위 과확장 위험**
   - 팬/줌 AI와 품질 AI를 동시에 시작하면 일정 지연 가능성 큼
   - 먼저 트랙1 합격 게이트를 고정해야 함

4. **평가 인프라 부족**
   - 장시간 표준 벤치마크 세트와 품질 스코어 정책 부재

## 7) 권장 다음 단계(구체)

## Phase A (즉시, 1~2주)

- 트랙1 오프라인 합격 기준 고정:
  - 장시간 완료
  - 리포트 필드 완결성
  - fail-safe 동작
- 벤치마크 실행기 + 요약 테이블 생성기 추가
- 캘리브레이션 프로파일(`calib.json`) 수명주기 관리 추가

## Phase B (다음, 2~4주)

- 트랙2 비AI 뷰포트 렌더러 구현:
  - 입력: 파노라마 프레임
  - 출력: `(cx, cy, zoom)` 기반 1080p crop
- 규칙 기반 카메라 정책 baseline + 스무딩 제약 추가

## Phase C

- 탐지/추적 + 하이브리드 정책(AI + 룰) 추가
- 룰 모드 강제 fallback 유지

## Phase D

- 라이브 입력 프로토타입 시작(엄격한 범위):
  - 2 스트림 고정
  - 재연결 + 지터 버퍼
  - 첫 라이브 마일스톤에서는 AI 품질 모듈 미포함

## 8) Go/No-Go 판단(현재 시점)

- **트랙1 오프라인 MVP:** Go
- **트랙2 AI 자동 팬/줌:** Go (단, 뷰포트 baseline과 정책 스캐폴딩 이후)
- **트랙3 AI 품질 보정:** 조건부 Go (오프라인 우선, 옵션 모듈)
- **지금 바로 라이브 운영 시도:** No-Go

## 9) 최종 권고

제안된 로드맵을 진행하되, 다음 게이트를 강제하세요.

1. 트랙1 신뢰성 게이트 먼저 통과
2. 트랙2는 AI 이전에 렌더러/룰 정책 완성
3. 트랙3는 옵션형 고도화로 분리
4. 라이브 전환은 처리량/복원력 목표 수치 충족 후 착수
